# Output Color Management: sRGB ICC Profile Embedding

## Problem

The RawAlchemyCpp pipeline produces JPEG and TIFF output files without any ICC profile or color space metadata. Viewers and image editors have no way to know the output is in BT.709/sRGB gamut, leading to potential color misinterpretation.

## Context

The LUT used in the pipeline (`FLog2C_to_CLASSIC-Neg._65grid_V.1.00.cube`) declares its output as:
- **Gamut**: ITU-R BT.709 (same primaries and white point as sRGB)
- **Gamma**: Classic Neg. film simulation curve (display-referred)

Since BT.709 and sRGB share identical chromaticities and D65 white point, no pixel-level color transformation is needed. The data is already display-ready. The only requirement is to embed an sRGB ICC profile to declare the color space.

## Design

### Approach: Hardcoded sRGB v2 ICC Profile

Embed the sRGB v2 ICC profile as a `constexpr uint8_t[]` in a header file. Both writers include this header and write the profile automatically.

### Files

#### New: `include/icc_srgb.h`

- Contains the sRGB v2 ICC profile (~3144 bytes) as a `constexpr` byte array
- Generated by script from a standard sRGB v2 ICC profile binary
- `SRGB_ICC_PROFILE[]` and `SRGB_ICC_PROFILE_SIZE` constants in `rawalchemy` namespace

#### Modified: `src/tiff_writer.cpp`

- Add `#include "icc_srgb.h"`
- In `setCommonTags()`, call `TIFFSetField(tif, TIFFTAG_ICCPROFILE, (uint32_t)SRGB_ICC_PROFILE_SIZE, SRGB_ICC_PROFILE)`
- No API changes to `writeTiff16()` or `writeTiff16Uncompressed()`

#### Modified: `src/jpeg_writer.cpp`

- Add `#include "icc_srgb.h"`
- Before `tj3Compress8()`, call `tj3SetICCProfile(handle, SRGB_ICC_PROFILE, SRGB_ICC_PROFILE_SIZE)`
- No API changes to `writeJpeg()`

### What We Do NOT Change

- **`ImageBuffer`**: No color space field added. ICC embedding is a write-layer concern.
- **Pipeline logic**: No gamut mapping, tone curve, or rendering intent steps added.
- **`main.cpp`**: No new CLI flags. Behavior is automatic.
- **C API**: No new parameters. Profile is embedded unconditionally for TIFF/JPEG output.

### Dependencies

No new dependencies. Both `TIFFTAG_ICCPROFILE` (libtiff) and `tj3SetICCProfile()` (libjpeg-turbo) are already available in the existing third-party libraries.

### Testing

- Verify TIFF output contains ICC profile: `tiffinfo output.tif | grep "ICC Profile"`
- Verify JPEG output contains ICC profile: `exiftool -ICC_Profile output.jpg`
- Visual validation: open output in color-managed viewer (e.g., Photoshop, Firefox) and confirm no color shift
